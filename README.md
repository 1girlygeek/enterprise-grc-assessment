# enterprise-grc-assessment

Enterprise GRC Assessment: Incident Investigation & Control Effectiveness Review

Overview

This respository demonstrates how a Technical Governance, Risk, and Compliance (GRC) analyst can evaluate operational incidents to identify control weakness, assess business risk, and recommend improvements that strengthen governance and reduce future risk.

The project follows a structured GRC assessment methodology using realistic healthcare SaaS scenarios. Rather than focusing only on technical troubleshooting, the assessment emphasizes governance, and executive decision support.

The scenarios, documentation, and artificats inthis repository were created for portfolio and educational purpose.

Project Objectives
- Investigate multiple operational and security incidents
- Identify governance and control deficiencies
- Evaluate the effectiveness of existinf controls
- Assess business and operational risk
- Map findings to industry-recognized frameworks
- Develop practical recommendations for leadership
- Demonstrate Technical GRC documentation and analysis skills

Assessment Methodology

Phase 1: Discovery
-Define objectives
-Collect evidence
-Review incidents
-Identify stakeholders
-Understand the business context

Phase 2: Assessment
-Analyze incidents
-Evaluate existing controls
-Perform root cause analysis
-Assess risks and business impact

Phase 3: Recommendations
-Identify control gaps
-Prioritize risks
-Recommend remediation
-Develop a roadmap

Phase 4: Reporting
-Executing summary
-Risk register
-Control effectiveness review
-Final recommendations
